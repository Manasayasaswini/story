To view the output, you first need to understand that the Sarvam API doesn't send you a "playable" link; it sends a **Base64-encoded string** that represents the audio. Your script needs to decode that string and write it to a `.wav` file.

Once the script finishes running, you can find the output in the `outputs/` folder you created. Here is the process to view and play it:

### 1. Where the file lives

Locate your project folder in your File Explorer (Windows) or Finder (Mac). Inside, you will see a folder named `outputs`. Inside that folder, you should find:

* `story_male.wav`
* `story_female.wav`

### 2. How to Play It

* **Standard Player:** Double-click the `.wav` files to play them in Windows Media Player, VLC, or QuickTime.
* **In VS Code:** If you are using VS Code, you can right-click the file and select "Reveal in File Explorer" to open it in your system's default player.

---

### 3. Troubleshooting (If the folder is empty)

If you don't see the files, your script might be receiving the data but not saving it correctly. Sarvam's API response usually looks like this:

```json
{
  "audios": ["UklGRue... (long base64 string)"]
}

```

**Ensure your script's logic looks like this to "see" the output:**

```python
import base64

# ... after getting the 'response' from the API ...
data = response.json()
audio_base64 = data['audios'][0]  # Get the first audio string
audio_bytes = base64.b64decode(audio_base64) # Decode it

with open("outputs/story_male.wav", "wb") as f:
    f.write(audio_bytes)
print("Success! View your file in the outputs/ folder.")

```

### 4. Direct Playback (Python Tip)

If you want to hear the audio immediately without opening a folder, you can add this to the end of your script:

```python
import os
# For Windows
os.startfile('outputs/story_male.wav')
# For Mac
# os.system('afplay outputs/story_male.wav')

```
